Multi-Armed Bandit (Sutton & Barto Chapter 2)

# 第2章 多臂混混机问题 (Multi-Arm Bandits)

# 基本概念

强化学习与其他类型学习最重要的区别是：它使用“评估式反馈”而非“指示式反馈”。

  • 评估式反馈：只告诉你选的这个动作“好不好”，但不告诉你是不是最好的
  • 指示式反馈：明确指出哪个动作是正确的，与是否执行无关

# 2.1 $n$ 臂混混机问题

  • 每次从 $n$ 个动作中选择一个
  • 每个动作对应一个固定但未知的奖励分布
  • 目标是最大化未来若干步内的累计期望奖励，例如 1000 步内

请考虑以下学习问题：
你反复面临从多个不同选项（或动作）中进行选择的问题。每次选择后，你会收到一个根据所选动作的固定概率分布生成的数值奖励。
你的目标是：最大化某一时间段内的期望总奖励，例如在 1000 次动作选择或时间步内。

这就是著名的 n 臂老虎机问题（n-armed bandit problem） 的原始形式。其名称来源于老虎机——又称为“单臂强盗（one-armed bandit）”，不过它拥有 n 个拉杆，而非一个。

每一个动作选择就像是在老虎机上拉一次拉杆，而奖励就是命中“头奖”后的收益。
通过多次重复的选择，你希望通过集中选择某些拉杆来最大化你的盈利。另一种类比是医生为多位重病患者选择治疗方案，每次选择一个治疗手段，而每次的“奖励”就是该患者的康复程度。今天，“n 臂老虎机问题”这个术语通常泛指此类问题的推广形式，但在本书中我们专指此种形式。

在我们的 n 臂老虎机问题中，每个动作都有一个期望奖励（mean reward），假设我们将某一动作的期望奖励称作该动作的“值（value）”。
若你知道每个动作的值，那么解决该问题是非常简单的：你总是选择值最大的动作即可。
但现实中我们并不知道各动作的值，我们只能逐步估计它们。

因此，若你在某一时刻估计的动作值中，至少有一个的估值最大，那么选择它即是一个贪婪动作（greedy action）。如果你选择这样的动作，即说明你正在**利用（exploiting）**你对当前动作值的已有知识。

反之，若你选择了一个非贪婪的动作，即说明你正在探索（exploring），以尝试获得对该动作值的更好估计。利用是从当前角度来看最优的选择，但探索可能带来更大的长期收益。
举例来说：假设贪婪动作的值是已知且确定的，而另一些动作的估值接近，但不确定性很大。那么至少有一个非贪婪动作可能实际上比贪婪动作更优，但你并不知道哪一个。因此如果你有足够多的时间步用于做选择，那么探索这些不确定的动作并找出更优的选择可能更有利。

短期来看探索带来的奖励可能较低，但长期来看，由于发现了更优的选择，你可能获得更大的收益。
由于无法在单次选择中既探索又利用，因此“探索 vs 利用”的**冲突（conflict）**成为强化学习中一个核心问题。

在某些特定情况下，是否该探索或利用依赖于估值的准确性、不确定性、剩余时间步数等因素的复杂组合。
虽然存在很多方法平衡探索与利用，例如对 n 臂老虎机问题和其推广问题的数学形式进行优化，但大多数方法假设环境是平稳的（stationary）且事先可知的，这在现实世界中往往不成立。

本书不会从过于复杂的角度来研究如何平衡探索与利用。我们甚至不会过分担心这种平衡，而是重点介绍几种简单有效的策略。这些方法远优于只利用、永远选择当前最优动作的方法。探索与利用的平衡是强化学习的典型挑战，而 n 臂老虎机的简单设置使我们能在清晰的框架中展现这一挑战。

# 2.2 动作价值方法（Action-Value Methods）与 ε-greedy 策略详解

我们接下来要更深入地研究一些**估计动作值（value）**的简单方法，并使用这些估计值来指导动作选择决策。

在本章中，我们用 $Q_t(a)$ 来表示第 $t$ 步时对动作 $a$ 的值（即平均奖励）的估计。
真实动作值是执行该动作时获得的期望奖励，而估计值就是对该期望的近似。

一种自然的估计方法是取每次选择该动作时获得的实际奖励的平均值。

换句话说，若在第 $t$ 步之前，动作 $a$ 被选择了 $N_t(a)$ 次，获得了奖励 $R_1, R_2, ..., R_{N_t(a)}$，那么对动作 $a$ 的值估计为：

$$ Q_t(a) = \frac{R_1 + R_2 + \cdots + R_{N_t(a)}}{N_t(a)} \tag{2.1} $$

# ✅ 等价递推更新公式

在实际实现中，我们不需要每次重新遍历所有过去的奖励 $R_i$ 来计算平均值，而可以使用一个在线更新公式（incremental update）：

Q[action] += (reward - Q[action]) / N[action]
1

这是 式 (2.1) 的递推更新形式。它允许我们在每次收到新奖励后，仅使用当前估计值和新奖励来更新，不需要保存所有历史数据。

# 📘 举例说明：为什么这两个公式是等价的？

# 环境设定：

我们有3个动作，每个动作的奖励是均匀分布如下：

动作 奖励分布
1 均匀分布于 [0, 1]
2 均匀分布于 [0, 2]
3 均匀分布于 [0, 3]

现在依次选择动作序列：[1, 2, 2, 3]

# 第一次选择动作 2：

  • 奖励：0.1
  • 更新次数：$N = 1$
  • 初始化 $Q = 0$

更新方式一（平均值）： $$ Q_1(2) = \frac{0.1}{1} = 0.1 $$

更新方式二（递推公式）：

Q[2] = 0 + (0.1 - 0) / 1 = 0.1
1

✅ 结果一致

# 第二次选择动作 2：

  • 奖励：0.9
  • 更新次数：$N = 2$

更新方式一（平均值）： $$ Q_2(2) = \frac{0.1 + 0.9}{2} = 0.5 $$

更新方式二（递推公式）：

Q[2] = 0.1 + (0.9 - 0.1) / 2 = 0.1 + 0.4 = 0.5
1

✅ 再次一致

# ✨ 小结

方法 计算方式 优点
平均公式 $Q_t(a) = \frac{\sum R_i}{N_t(a)}$ 每次计算需保存所有历史奖励 理论直观
递推公式 $Q \mathrel{+}= \frac{\text{reward} - Q}{N}$ 每次只需当前值和次数 高效实用、节省内存

✅ 两者数学上完全等价，后者更适合实际强化学习中的实现，尤其是处理大规模动作空间时。

📚 使用 Sample Average 的动作选择方法

如果 $N_t(a) = 0$，那么我们可以定义 $Q_1(a)$ 为某个默认值，例如 $Q_1(a) = 0$。随着 $N_t(a) \to \infty$，根据大数定律，$Q_t(a)$ 会收敛到 $q(a)$。我们将这种方法称为 sample-average method（样本均值法），因为每次估计值是该动作相关奖励的简单平均。

当然这只是众多估计动作值的方法之一，并非最优方法。尽管如此，我们暂且从这个简单方法开始，接下来讨论如何使用这些估计值来选择动作。

🤖 贪婪与 ε-贪婪动作选择策略

最简单的动作选择策略是：选择当前具有最高估计值的动作（或之一）。也就是在第 $t$ 步选择：

$$ A_t = \arg\max_a Q_t(a) \tag{2.2} $$

其中 $\arg\max_a$ 表示在所有动作 $a$ 中选取使得表达式取最大值的动作（如遇平局则随机选择）。该策略称为 贪婪策略（greedy action selection）。

这种策略完全基于当前知识做出最大化预期奖励的选择，没有探索其他看似较差但可能实际更好的动作。

一个替代策略是使用 ε-贪婪方法（ε-greedy）： • 在绝大多数时间（概率 $1 - \varepsilon$）选择当前估计值最高的动作 • 在少数时间（概率 $\varepsilon$）随机从所有动作中选择一个

这种方式会确保每个动作最终都会被采样无穷次，从而满足 $Q_t(a) \to q(a)$，即估计值最终会收敛到真实值。

这意味着选择最优动作的概率也会收敛到接近 $1 - \varepsilon$，即接近确定性。

这些只是渐近性质，实际效果还需实证分析。

🔬 实验：贪婪 vs ε-贪婪 策略的表现

为粗略评估贪婪与 ε-贪婪方法的效果，我们在一组测试问题上进行了实验： • 共进行了 2000 次实验，每次为一个 10 臂老虎机任务（$n = 10$） • 每个动作的真实值 $q_(a)$ 从均值为 0、方差为 1 的正态分布中抽取 • 每一步执行一个选择动作 $A_t$，其奖励 $R_t$： • 均值为 $q_(A_t)$，加上均值为 0、方差为 1 的噪声

我们比较了三种策略： • 贪婪（ε = 0） • ε-贪婪，ε = 0.01 • ε-贪婪，ε = 0.1

图表显示了三种策略在 1000 步以内的平均表现（在 2000 个独立任务上取平均）： • 上图：平均奖励 vs 步数 • 下图：选择最优动作的比例 vs 步数

# 更新 Q 值（sample average）

def update_sample_average(Q, N, action, reward):
    N[action] += 1
    Q[action] += (reward - Q[action]) / N[action]
    return Q, N

# ε-greedy 动作选择

import numpy as np

def epsilon_greedy(Q, epsilon):
    if np.random.rand() < epsilon:
        return np.random.randint(len(Q))  # 探索
    else:
        return np.argmax(Q)  # 利用

import numpy as np
import matplotlib.pyplot as plt

def run_bandit(epsilon, steps=1000, runs=2000, k=10):
    rewards = np.zeros((runs, steps))
    optimal_action_counts = np.zeros((runs, steps))

    for run in range(runs):
        q_true = np.random.normal(0, 1, k)
        optimal_action = np.argmax(q_true)
        Q = np.zeros(k)
        N = np.zeros(k)

        for t in range(steps):
            action = epsilon_greedy(Q, epsilon)
            reward = np.random.normal(q_true[action], 1)

            Q, N = update_sample_average(Q, N, action, reward)

            rewards[run, t] = reward
            if action == optimal_action:
                optimal_action_counts[run, t] = 1

    return rewards.mean(axis=0), optimal_action_counts.mean(axis=0)

epsilons = [0, 0.01, 0.1]
colors = ['green', 'red', 'black']

plt.figure(figsize=(12, 5))

plt.subplot(1, 2, 1)
for eps, c in zip(epsilons, colors):
    avg_reward, _ = run_bandit(eps)
    plt.plot(avg_reward, label=f'ε = {eps}', color=c)
plt.title("Average Reward")
plt.xlabel("Steps")
plt.ylabel("Average reward")
plt.legend()

for eps, c in zip(epsilons, colors):

plt.legend()

# ✅ 实现参考

def simulate_bandit(k: int, actions: list[int], rewards: list[float], epsilon: float, seed: int) -> list[float]:
    np.random.seed(seed)
    Q = [0.0] * k
    N = [0] * k

    for a, r in zip(actions, rewards):
        N[a] += 1
        Q[a] += (r - Q[a]) / N[a]

    return Q

# 状态价值函数 V(s) 与 动作价值函数 Q(s, a) 的区别与 TD 更新对比

# 🧠 V 和 Q 的根本区别

概念 状态价值函数 V(s) 动作价值函数 Q(s, a)
表示 在状态 s 下的期望回报 在状态 s 下执行动作 a 后的期望回报
使用场景 策略评估（如 TD, Monte Carlo） 策略改进、控制（如 Q-learning, SARSA）
依赖 仅状态 s 状态 s 和动作 a
代表的是谁的期望 当前策略 π 在状态 s 下的表现 当前策略 π 在状态 s 做出 a 的表现

# 🔁 TD 更新（V值）

时序差分法（Temporal-Difference, TD）用于学习状态价值函数 V(s)：

$$ V(s_t) \leftarrow V(s_t) + \alpha \left[ r_{t+1} + \gamma V(s_{t+1}) - V(s_t) \right] $$

  • 学的是：当前策略下，状态 s_t 的长期期望回报。
  • TD 是 on-policy 策略评估方法。

# 🆚 Q-learning 更新（Q值）

Q-learning 学的是最优动作价值函数：

$$ Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ r_{t+1} + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \right] $$

  • max_a' 表示从下一个状态中选择最优动作；
  • 这是用于策略优化的 off-policy 控制方法；
  • 用于动作选择，如 ε-greedy 策略。

# 🔍 类比理解

场景 V(s) Q(s, a)
你在某个城市 V(s) 告诉你这个城市的“幸福指数” Q(s, a) 告诉你在这个城市“选择某种活动”后未来几天可能有多开心
投资决策 V(s) 是当前市场状态的收益期望 Q(s, a) 是在当前市场状态选择某种投资方案的期望收益

# ✅ 总结对比表

维度 V(s)（TD学习） Q(s, a)（Q-learning / ε-greedy）
输入 状态 s 状态 s 与动作 a
输出 s 的价值估计 s, a 对的价值估计
用途 评估策略 控制策略（选择动作）
TD 更新 基于 V(s') 基于 max Q(s', a') 或 Q(s', a')
算法例子 TD(0), TD(λ), Monte Carlo SARSA, Q-learning, DQN

# V(s) 与 Q(s, a) 是否能解决相同问题？——全面解析

# ✅ 结论

V(s) 和 Q(s, a) 都可以用来解决相同的问题——也就是找到最优策略（optimal policy），但它们的方式不同，适用的算法也不同。

# 🎯 相同目标：找到最优策略 π*

无论使用 V 还是 Q，我们最终的目标都是：

找到一个策略 π，使得在每个状态下都能获得最大的长期累计回报。

# 🔍 核心区别：策略表示与学习方式不同

项目 基于 V(s) 的方法 基于 Q(s, a) 的方法
策略获取方式 π(s) 需通过额外搜索（如策略迭代） π(s) = argmaxₐ Q(s, a) 可直接决策
学的是 状态的价值 状态+动作的组合价值
策略学习 需配合策略评估与改进 Q 值本身即为控制策略依据
典型算法 TD(0), TD(λ), Policy Iteration Q-learning, SARSA, DQN

# 🧠 实例对比：迷宫导航

# 状态 s：你当前的位置

# 动作 a：上下左右移动

# ✅ 基于 V(s)

  1. 用 TD(0) 或策略迭代估计 V(s)；
  2. 每一步用 one-step lookahead 搜索： $$ a^* = \arg\max_a \left[ r + \gamma V(s') \right] $$
  3. 依赖环境模型（环境转移概率）或模拟器。

# ✅ 基于 Q(s, a)

  1. 直接估计每个状态-动作对的价值；
  2. 策略就是： $$ \pi(s) = \arg\max_a Q(s, a) $$
  3. 完全 model-free，可用于实际游戏、控制。

# ✅ 总结对比

比较点 V(s) 方法 Q(s, a) 方法
是否能解决相同问题 ✅ 是 ✅ 是
方法是否相同 ❌ 否 ❌ 否
是否适合策略评估 ✅ 是 ❌ 通常用于控制
是否适合控制 ⚠️ 不直接适合 ✅ 是
是否需要环境模型 ⚠️ 通常需要 ✅ 不需要（model-free）

# 2.3 增量实现

增量平均更新： $Q_{k+1} = Q_k + \frac{1}{k}(R_k - Q_k)$

通用形式： $\text{NewEstimate} \leftarrow \text{OldEstimate} + \alpha (\text{Target} - \text{OldEstimate})$

# 2.4 非平稳问题追踪 Tracking a Nonstationary Problem

指数衰减平均（constant step-size）： $Q_{k+1} = Q_k + \alpha (R_k - Q_k)$

展开后形式： $Q_{k+1} = \alpha R_k + (1-\alpha)\alpha R_{k-1} + \cdots + (1-\alpha)^k Q_1$

# 2.5 乐观初值

  • 设置初值为较大数（如 $Q_1(a) = 5$）
  • 初始阶段鼓励尝试所有动作

# 2.6 上置信界（UCB）方法

选择规则： $A_t = \arg\max_a \left( Q_t(a) + c \sqrt{\frac{\ln t}{N_t(a)}} \right)$

# 2.7 梯度策略（Gradient Bandits）

策略概率（softmax）： $\pi_t(a) = \frac{e^{H_t(a)}}{\sum_b e^{H_t(b)}}$

偏好更新规则：

  • 对选择动作： $H_{t+1}(A_t) = H_t(A_t) + \alpha (R_t - \bar{R_t})(1 - \pi_t(A_t))$
  • 对未选择动作： $H_{t+1}(a) = H_t(a) - \alpha (R_t - \bar{R_t})\pi_t(a)$

# 2.8 上下文混混机问题（Contextual Bandits）

  • 每个轮次给出提示，例如不同颜色，表示不同的 task
  • 可学习从“情景”到“动作”的策略：即学习 policy

# 2.9 总结（Summary）

  • ε-greedy：简单、有效，每个动作最终都会被尝试无穷次
  • UCB：确定性地选择最不确定的动作，避免无效探索
  • 梯度 Bandit：不估计值，而估计偏好，用 softmax 概率决策
  • 乐观初始化：即使 greedy 方法也能通过初始化值强迫探索

def greedy_bandit(bandit, n):
    Q = [0.0] * n
    N = [0] * n
    for _ in range(1000):
        a = argmax(Q)
        r = bandit(a)
        N[a] += 1
        Q[a] += (1 / N[a]) * (r - Q[a])
    return Q

def weighted_average(rewards, alphas):
    total = 0
    weight = 1
    for r, alpha in zip(reversed(rewards), reversed(alphas)):
        weight *= (1 - alpha)
        total += weight * r
    return total
